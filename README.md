# 🧠 AI Sales Forecasting System

> **An end-to-end intelligent sales analytics dashboard** that combines PostgreSQL data warehousing, real-time Streamlit visualizations, and AI-powered strategic insights using Groq's LLaMA 3.3 70B model.

---

## ✨ Key Features

| Feature | Description |
|---------|-------------|
| 📊 **Interactive Dashboard** | Streamlit-powered UI with KPI cards, spline charts, and date-range filters |
| 🗄️ **PostgreSQL ETL Pipeline** | Blazing-fast bulk data loading via `COPY` with multi-table JOIN into a master view |
| 🤖 **AI Strategy Reports** | One-click business intelligence reports generated by LLaMA 3.3 70B via Groq API |
| 📅 **Flexible Date Selection** | Dropdowns for start/end dates (2013–2017) with quick-select buttons (3M / 6M / 1Y) |
| 📈 **Growth Analytics** | Automatic period-over-period growth calculation with trend detection |
| 🔍 **Prompt Engineering** | Dynamic prompt injection — real computed metrics fed into a structured AI prompt template |

---

## 🖥️ Dashboard Preview

The dashboard presents four key metric cards (Total Sales, Growth %, Avg Transactions, Trend), a daily sales trajectory chart, a database explorer, and an AI-generated strategic report — all updated in real time based on user selections.

## 🚀 Getting Started

### Prerequisites

- **Python 3.9+**
- **PostgreSQL** installed and running
- **Groq API Key** — Get one free at [console.groq.com](https://console.groq.com)

### 1. Clone the Repository

```bash
git clone https://github.com/bhaveshksh/AI-Sales-Forecasting-System.git
cd AI-Sales-Forecasting-System
```

### 2. Create Virtual Environment

```bash
python -m venv ai_sales
ai_sales\Scripts\activate        # Windows
# source ai_sales/bin/activate   # macOS/Linux
```

### 3. Install Dependencies

```bash
pip install -r requirements.txt
```

### 4. Configure Environment Variables

Create a `.env` file in the project root:

```env
GROQ_API_KEY=your_groq_api_key_here
DB_USERNAME=postgres
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432
DB_NAME=postgres
```

### 5. Download Datasets

Download the datasets from the [Kaggle — Store Sales: Time Series Forecasting](https://www.kaggle.com/competitions/store-sales-time-series-forecasting/data) competition and place them in the `Datasets/` folder.

> **Note:** `train.csv`, `test.csv`, and `sample_submission.csv` are excluded from the repository due to their large file sizes (>100 MB).

### 6. Run the ETL Pipeline (One-Time Setup)

```bash
python database_setup.py
```

This will:
- Create the `sales_forecasting` schema in PostgreSQL
- Load all CSV files into individual tables
- Create the `master_data` table via a 4-way JOIN

### 7. Launch the Dashboard

```bash
streamlit run app.py
```

The app will open at `http://localhost:8501`.

## 📊 How It Works

1. **Select a Store** — Choose from 54 retail stores via the sidebar dropdown
2. **Pick a Date Range** — Use the start/end date dropdowns or quick-select buttons (Past 3M / 6M / 1Y)
3. **View Analytics** — KPI cards and sales trajectory chart update instantly

## 🔑 Key Design Decisions

- **PostgreSQL over raw CSV** — 3M+ rows need persistent, indexed storage for fast queries
- **`COPY` over `INSERT`** — Native bulk loading is 10–100x faster for large datasets
- **Groq over OpenAI** — Faster inference (LPU hardware), generous free tier, competitive model quality
- **`temperature=0.3`** — Low temperature ensures consistent, factual business reports
- **`@st.cache_data`** — Caches the 3M+ row query so the UI stays snappy across interactions
